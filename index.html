What is Angular?

  • Developed by Google
  • An open source JavaScript based framework for building client applications in HTML
  • Javascript or Typescript can be used
  • Composes HTML Templates and Angularized markup
  • Empowers developers to build applications that live on the web, mobile, or the desktop

Introduction of Typescript

  • Superset of JavaScript (ES6) – but can compile till ES3
  • Productive in writing JS application
  • Implement the OOPS Concept
  • A Java or C# developer can easily code in Angular. You don’t have to be a JavaScript developer.
  • Lambda operators, iterators or reflection’s mechanism is available
  • Optional Types and Arrow Functions are available

Angular Architecture

Core Concepts ( Modules)

  • Modules – Angular apps are modular and Angular has its own modularity system called NgModules.
  • NgModule – Decorator that takes the single metadata object

										import { NgModule }      from '@angular/core';
										import { BrowserModule } from '@angular/platform-browser';
										@NgModule({
										  imports:      [ BrowserModule ], 
										  providers:    [ Logger ],
										  declarations: [ AppComponent ],
										  exports:      [ AppComponent ],
										  bootstrap:    [ AppComponent ]
										})export class AppModule { }

Core Concepts (Angular libraries)

  • Library Modules - Angular ships as a collection of JavaScript modules.
  • Each Angular library name begins with the @angular prefix.
  • For example

										// angular component decorator
										import { Component } from '@angular/core'; 
										import { BrowserModule } from '@angular/platform-browser';

Core Concepts(Components)

  • Most basic building block
  • An Angular application is a tree of Angular components.
  • Must belong to NgModule
  • Metadata configuration specified via the Component decorator
  • Angular creates, updates, and destroys components as the user moves through the application with use of various lifecycle hooks.
